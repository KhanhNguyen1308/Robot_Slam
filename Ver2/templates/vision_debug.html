Disparity & Depth Estimation
Tính toán disparity map (độ lệch pixel giữa 2 ảnh stereo) bằng StereoBM. Disparity tỷ lệ nghịch với khoảng cách: disparity cao = gần camera, disparity thấp = xa camera. Công thức depth: Z = (focal_length × baseline) / disparity

4
FAST Feature Detection
FAST (Features from Accelerated Segment Test) phát hiện corner points (góc) trong ảnh. Thuật toán kiểm tra 16 pixel xung quanh một điểm: nếu có ≥12 pixel liên tiếp sáng hơn hoặc tối hơn ngưỡng → đó là corner. Rất nhanh, phù hợp cho real-time processing.
FAST Keypoints (green circles)
5
ORB Feature Descriptors
ORB (Oriented FAST and Rotated BRIEF) kết hợp FAST với BRIEF descriptor.
BRIEF (Binary Robust Independent Elementary Features): Mô tả feature bằng binary string (256-512 bits) bằng cách so sánh cường độ sáng của các cặp pixel xung quanh keypoint.
ORB thêm orientation (hướng) để BRIEF trở nên rotation-invariant. Màu đỏ = keypoint với orientation (đường kẻ chỉ hướng).
ORB Keypoints with Orientation (red)
6
Hamming Distance Matching
So khớp features giữa ảnh trái và phải bằng Hamming distance. Hamming distance = số lượng bit khác nhau giữa 2 binary descriptors.
Ví dụ: descriptor A = 10110, B = 10010 → Hamming distance = 1 (bit thứ 3 khác nhau)
BFMatcher (Brute Force Matcher) tìm best match cho mỗi feature trong ảnh trái. Đường nối màu xanh = good matches (top 20 matches với distance thấp nhất).
